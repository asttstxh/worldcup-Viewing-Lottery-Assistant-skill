# Contributing

欢迎改进这个世界杯赛前分析 skill。优先接受能提升信息可靠性、报告可读性、响应式体验和负责任购彩提示的贡献。

## 贡献方向

- 增强数据源优先级，例如补充官方赛程、国家队公告、体育彩票公开信息或更可靠的球员照片来源。
- 优化 `references/report-spec.md`，让新报告更稳定地覆盖阵容、战术、赔率、风险和预算分配。
- 改进 HTML 示例的移动端展示、球员卡交互、战术板和预算进度条。
- 增加更多示例报告，但请确保示例不包含本地路径、私人信息、账号信息或无法验证的传闻。
- 改进安装脚本和文档，让首次使用者能更顺利地安装。

## 基本要求

- 所有事实性内容都应尽量有来源。
- 赛程、阵容、伤病、赔率、教练、排名、身价和彩票规则属于易变化信息，必须在生成报告时实时核验。
- 推断内容必须明确标注为推断，不得写成确定事实。
- 不得宣传保证中奖、稳定盈利或无风险回报。
- 面向用户的中文文案应保持清晰、正式、自然，避免网络化表达。

## 本地检查

修改 skill 后，建议运行 Codex skill 校验：

```bash
python3 ~/.codex/skills/.system/skill-creator/scripts/quick_validate.py skills/worldcup-match-report
```

如果你的环境没有该脚本，请至少检查：

- `skills/worldcup-match-report/SKILL.md` 有合法 YAML frontmatter。
- skill 名称为 `worldcup-match-report`。
- `references/report-spec.md` 中的报告契约与示例 HTML 保持一致。
- 示例报告在桌面和手机宽度下没有文字重叠、横向溢出或不可点击的球员卡。

## 负责任使用

这个项目服务于赛前信息整理和娱乐性预算规划。请不要把它改造成承诺收益、规避风险提示或诱导超预算购彩的工具。
